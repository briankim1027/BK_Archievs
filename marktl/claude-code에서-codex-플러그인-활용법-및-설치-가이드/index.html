    요약

    codex-plugin-cc는 Claude Code에서 Codex의 리뷰와 작업 위임 기능을 연결해주는 공식 플러그인입니다.

    연결 방식

    기존 Codex CLI와 app server를 활용해 Claude Code 안에서 Codex 명령어를 사용할 수 있습니다.

    중심 사용처

    코드 리뷰, 2차 검토, 일부 작업 위임 등 워크플로우 효율화에 초점이 있습니다.

    도입 난이도

    설치와 인증이 비교적 단순하며, 상태 확인과 결과 조회 명령도 제공됩니다.

    설치 방법

    설치는 마켓플레이스 추가, 플러그인 설치, 로드 후 설정의 순서로 진행됩니다.

    1
    마켓플레이스 추가 
    /plugin marketplace add openai/codex-plugin-cc
    2
    플러그인 설치 
    /plugin install codex@openai-codex
    3
    로드 후 설정 
    /reload-plugins

/codex:setup
    수동 설치와 로그인 명령 보기 
    npm install -g @openai/codex

!codex login

    /codex:setup은 Codex 준비 상태를 확인하며, Codex가 설치되어 있지 않고 npm을 사용할 수 있으면 설치를 제안할 수 있습니다.

    04

    어떻게 동작하나요

    이 플러그인은 Claude Code가 자체적으로 Codex처럼 변신하는 구조가 아니라, 로컬 Codex CLI와 Codex app server를 통해 위임하는 구조입니다. 

    Claude Code
  ↓
Codex Plugin
  ↓
로컬 Codex CLI
  ↓
Codex app server
  ↓
Codex 실행 및 결과 반환
    재사용되는 것

    로컬 인증, 기존 Codex config, MCP 설정, base URL 설정을 재사용할 수 있습니다.

    얻는 효과

    Claude Code 사용자가 새로운 환경을 열지 않고 Codex 기능을 워크플로우 안에 끼워 넣을 수 있습니다.

    특징설명
    별도 런타임 아님이미 쓰고 있는 Codex 설정을 재사용
    로컬 인증 재사용기존 ChatGPT 로그인 또는 API key 설정 사용
    환경 재사용Codex config, MCP 설정, base URL 설정 반영 가능
    진입 장벽 감소Claude Code 사용자에게 Codex를 얹는 방식

    05

    현재 확인되는 주요 명령어

    명령어설명성격
    /codex:review일반적인 읽기 전용 코드 리뷰리뷰
    /codex:adversarial-review더 비판적이고 공격적인 관점의 리뷰리뷰
    /codex:rescueCodex에게 작업을 넘김위임
    /codex:status백그라운드 작업 상태 확인관리
    /codex:result작업 결과 확인관리
    /codex:cancel작업 취소관리
    Finding 가장 주목받는 명령은 /codex:review와 /codex:adversarial-review입니다. 그래서 초기 인상은 리뷰 플러그인에 가깝습니다.

    “리뷰 전용인가요?”에 대한 정확한 정리

    왜 리뷰 중심으로 보이나

    공개 문맥과 사용자 반응, 첫인상은 대부분 코드 리뷰 플러그인에 가깝습니다.

    하지만 위임도 있음

    공식 README는 review, adversarial-review, rescue, status, result, cancel을 명시합니다.

    가장 안전한 표현

    “리뷰와 일부 작업 위임을 지원한다”가 공식 README 기준 가장 정확합니다.

    표현적절성
    완전한 구현 자동화 도구다과장 가능성 큼
    리뷰 중심 플러그인이다대체로 안전
    리뷰와 일부 작업 위임을 지원한다공식 README 기준 가장 정확

    표현 정확성 리뷰

    이 문서의 톤이 홍보보다 검증에 가깝게 유지되고 있나요?

    07

    각 명령은 언제 쓰면 좋은가요

    /codex:review

    PR 전 1차 검토, 리팩터링 이후 회귀 위험 체크, 빠른 코드 스멜 탐지에 적합합니다.

    /codex:adversarial-review

    보안 취약점, 예외 처리 누락, 논리 결함, edge case를 적극적으로 찾을 때 유용합니다.

    /codex:rescue

    작업 위임용 명령으로 소개되지만, 작은 저장소나 보조 작업에서 먼저 검증하는 편이 좋습니다.

    status/result/cancel

    백그라운드 작업을 확인하고, 결과를 가져오고, 필요할 때 중단하는 관리 흐름입니다.

    비유적 판단 기준 일반 리뷰가 “괜찮은지 보자”라면, adversarial review는 “무너질 지점을 찾아보자”에 가깝습니다.

    사용 시나리오 리뷰

    각 명령의 추천 상황이 실제 팀 운영 판단에 충분히 구체적인가요?

    08

    실제 사용 예시

    예시 1. PR 전 1차 리뷰 
    /codex:review --background

/codex:status

/codex:result

    먼저 돌려두고 결과를 확인하는 방식이 가능합니다.

    예시 2. 취약점 의심 코드 점검 
    /codex:adversarial-review

    인증/인가 로직, 결제 처리 분기, 파일 업로드, SQL 조합, 외부 입력 처리 변경에 유용합니다.

    예시 3. 작업 위임 흐름 실험 
    /codex:rescue

/codex:status

/codex:result

    초기 공개 단계에서는 핵심 기능보다 보조성 작업이나 2차 검토성 작업부터 붙이는 편이 현실적입니다.

    개발자 입장에서 왜 중요한가요

    핵심은 Claude와 Codex의 승패가 아니라, 서로 다른 에이전트를 한 워크플로우 안에서 섞어 쓰는 방식이 공식화되고 있다는 점입니다.

    컨텍스트 전환 감소

    Claude Code에서 작업한 뒤 별도로 Codex를 열고 결과를 가져오는 왕복 비용이 줄어듭니다.

    작성과 검토 분리

    설계와 대화, 1차 리뷰, 공격적 검토, 최종 수정을 역할별로 나누기 쉬워집니다.

    조합의 시대

    최고의 AI 하나보다 각 도구를 어디에 배치하느냐가 더 중요해졌다는 신호입니다.

    단계도구
    설계 및 대화Claude Code
    1차 리뷰Codex review
    공격적 검토Codex adversarial-review
    최종 수정개발자

    전략적 의미 리뷰

    이 섹션이 도구 비교 구도를 피하고 워크플로우 설계 관점으로 충분히 이동했나요?

    지금 시점에서 확인되는 한계

    자유 질의 명령 부재 현재는 /codex:ask 같은 범용 질의형 인터페이스보다 목적형 명령 세트에 가깝습니다.
    sandbox 관련 리뷰 오류 이슈 설치 직후 /codex:review 실행 시 sandbox variant 관련 오류가 보고된 바 있어 환경별 검증이 필요합니다.
    적합한 범위 PR 전 코드 리뷰, 보안성/논리성 2차 검토, 배경 작업 결과 조회처럼 가볍고 명확한 워크플로우에 잘 맞습니다.
    적합한 용도덜 적합한 용도
    PR 전 코드 리뷰무거운 범용 질의 세션
    보안성/논리성 2차 검토복잡한 장기 협업 대화
    배경 작업 결과 조회모든 개발 과정을 이 플러그인 하나로 대체
